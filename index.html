Research focus

Modern cloud systems fail in ways that are hard to predict and harder to recover from — misconfigurations, cross-system interactions, and cascading faults. I want to make these systems observably reliable, and to study whether autonomous agents can take on the operational work of keeping them healthy.

/01 · reliability

System reliability

Understanding and reducing failures at cloud and datacenter scale — configuration errors, fault recovery, and the interactions between services that cause real-world outages.

/02 · agents

Agentic SRE

Can LLM-driven agents diagnose, remediate, and safely operate production infrastructure? Bridging MLOps and site reliability to make automation trustworthy under failure.

/03 · practice

From practice to research

Hands-on infrastructure — Kubernetes, GitOps, Terraform, CI/CD — grounds my research in the failure modes operators actually face, not abstractions.

Open-source contribution

SREGym · agentic-SRE benchmark

Reproducing a real GKE IP-exhaustion outage as a reliability benchmark

I contributed a new benchmark problem to SREGym — the open benchmark for whether AI agents can resolve production incidents, from Prof. Tianyin Xu’s group at UIUC. It faithfully recreates a real-world GKE “ran out of IP addresses” outage as a reproducible Kubernetes fault, so agents can be scored on diagnosing and mitigating it.

  • Engineered a reliable pod IP-exhaustion fault with Calico IPAM — a runtime exhaustible IP pool plus strict affinity — that triggers without any cluster-wide reconfiguration, after iterating through four designs.
  • Made it push-button and safe to re-run: a Kind + Calico setup path and a global cleanup that restores cluster networking even if a run exits mid-fault.
  • Evaluated an LLM agent (Stratus + GPT-5) against the fault, surfaced a cross-namespace reasoning gap, and tuned the mitigation so the agent locates and scales down the offending workload to recover the app.

Real incident
GKE ran out of IPs. A /16 subnet (65,536 addresses) exhausted as nodes pre-allocated IP blocks — the autoscaler could no longer add nodes.
↓
Benchmark fault
Calico IP-pool exhaustion. A tiny pool is drained at runtime, leaving Hotel-Reservation pods stuck in ContainerCreating — no cluster changes required.
↓
Agent task
Diagnose the outage and mitigate it by finding and scaling down the workload eating the IPs.
